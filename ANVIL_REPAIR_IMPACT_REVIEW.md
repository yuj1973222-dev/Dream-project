# Anvil Repair Impact Review

작성일: 2026-06-04 KST

## 목적

`LeeSeolCrafting`의 돈 기반 수리 GUI를 도입하기 전에, 바닐라 모루 수리를 차단할 경우 어떤 기능이 같이 영향을 받는지 정리한다.

## 결론

모루 사용은 바로 전면 차단하지 않는다.

이유:

- 모루는 단순 수리뿐 아니라 이름 변경, 인챈트 책 합성, 장비 합성에도 사용된다.
- AdvancedEnchantments와 충돌 가능성이 있다.
- 기존 유저 장비의 이름 변경/인챈트 관리 루프가 막힐 수 있다.

우선은 돈 기반 수리 GUI를 추가하고, 모루 수리는 로그/테스트 후 제한 범위를 결정한다.

## 영향 범위

| 기능 | 전면 차단 시 영향 | 권장 처리 |
|---|---|---|
| 장비 수리 | 차단됨 | 돈 기반 수리 GUI로 대체 가능 |
| 아이템 이름 변경 | 같이 막힐 수 있음 | 별도 허용 여부 결정 필요 |
| 인챈트 책 합성 | 같이 막힐 수 있음 | 바로 차단 비추천 |
| 같은 장비 합성 | 같이 막힘 | 장비 성장 루프와 충돌 가능 |
| AdvancedEnchantments 적용/합성 | 충돌 가능 | paid plugin 설정 검토 후 판단 |
| 기존 유저 장비 | 수리 경로 변경 | 공지/보상/전환 정책 필요 |

## 구현 옵션

### A안: 모루 전면 차단

장점:

- 돈 기반 수리 시스템으로 경제 회수가 확실해진다.

단점:

- 이름 변경, 인챈트 합성까지 막을 위험이 크다.
- 유저 불편이 크다.

판정: 비추천.

### B안: 모루 수리만 차단

가능한 방식:

- `PrepareAnvilEvent` 또는 `InventoryClickEvent`에서 결과 아이템을 검사한다.
- 결과 아이템의 내구도 회복이 감지되는 경우만 차단한다.
- 이름 변경만 있는 경우는 허용한다.

주의:

- Paper/Minecraft 버전별 모루 결과 계산 방식이 달라질 수 있다.
- AdvancedEnchantments 커스텀 인챈트가 붙은 아이템은 ItemMeta/PDC 보존 검증이 필요하다.

판정: 2차 구현 후보.

### C안: 모루 유지 + 돈 기반 수리 GUI 병행

장점:

- 기존 플레이 흐름을 깨지 않는다.
- 먼저 수리 GUI 사용성을 검증할 수 있다.

단점:

- 경제 회수 효과가 약하다.

판정: 1차 구현 권장.

## LeeSeolCrafting 1차 권장 정책

```yaml
repair:
  enabled: true
  block-vanilla-anvil-repair: false
  log-anvil-repair: true
```

1차:

- 돈 기반 수리 GUI 구현
- 모루 수리 차단 안 함
- 모루 사용 로그만 선택적으로 기록

2차:

- 실제 유저 사용량 확인
- AdvancedEnchantments 충돌 확인
- 이름 변경/인챈트 합성/수리 구분 가능 여부 확인
- 필요한 경우 수리만 차단

## 검증 기준

| 테스트 | 기대 결과 |
|---|---|
| 손상 장비를 수리 GUI로 수리 | 돈 차감, 내구도 회복 |
| 돈 부족 상태에서 수리 | 내구도 변화 없음 |
| 모루 이름 변경 | 1차에서는 허용 |
| 모루 인챈트 책 합성 | 1차에서는 허용 |
| AdvancedEnchantments 아이템 수리 | ItemMeta/PDC/인챈트 유지 |
| 서버 재시작 | 수리 설정 유지 |
