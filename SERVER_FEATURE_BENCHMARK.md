# SERVER_FEATURE_BENCHMARK.md

2026-06-11 기준 유명 Minecraft 서버 기능 벤치마크. 이 문서는
`서버-운영-메인` 채팅에서 다음 개발 방향을 정할 때 참고한다.

## 비교 대상

- Hypixel SkyBlock: 성장, 경제, 경매/시장, 던전, 컬렉션, 스킬 구조 참고.
- Wynncraft: 퀘스트형 MMORPG, 직업/클래스, 던전/레이드, 지역 탐험 구조 참고.
- MCC Island: 일일/주간 퀘스트, 이벤트 패스, 코스메틱, 파티/팀 플레이,
  반복 플레이 보상 구조 참고.

## 우리 서버 현재 강점

- 이미 Velocity 기반 로비/야생/신월드 구조가 있다.
- 자체 플러그인이 많다: 경제, 직업, 랭크, 퀘스트, 제작, 경매, 던전, 전투,
  HUD, 마을/국가.
- ItemsAdder/리소스팩, TAB, DeluxeMenus 기반 시각 요소가 이미 있다.
- 돈, 랭크, 직업, 제작, 퀘스트가 서로 연결될 수 있는 구조가 있다.
- 2026-06-11 기준 `LeeSeolJobs`는 고정 직업 선택이 아니라 활동/탐험 보상
  레이어로 방향을 바꿨다.

## 냉정한 약점

- 기능은 많지만 플레이어가 따라갈 "큰 루프"가 아직 약하다.
- 퀘스트는 튜토리얼 중심이고, 서버의 장기 목표를 안내하지 못한다.
- Jobs, Crafting, Ranks, Dungeon, Combat 보상이 한 경제 안에서 아직 충분히
  묶이지 않았다.
- 고정 직업식 성장으로 가면 마인크래프트 자유도를 낮출 수 있으므로,
  활동/컬렉션/탐험 기록 중심으로 풀어야 한다.
- 아이템은 "획득처, 사용처, 소모처, 희귀도" 표가 먼저 필요하다.
- 던전은 기능 뼈대가 있지만 반복 플레이 이유가 아직 약하다.
- 경매는 있으나 시장/거래 경제의 중심 기능으로 쓰기에는 통계, 수수료,
  검색, 즉시구매 같은 운영 장치가 부족하다.

## 추가하면 좋은 기능 후보

### 1. 서버 가이드형 진행 패스

우선순위: 매우 높음.

MCC Island의 일일/주간 퀘스트와 이벤트 패스 구조처럼, 플레이어가 접속할 때
"오늘 할 일"과 "이번 시즌 목표"를 바로 알 수 있어야 한다.

우리 서버 적용:

- `LeeSeolQuest`를 튜토리얼 전용이 아니라 운영 가이드 패스로 확장한다.
- 일일 퀘스트: 광질 50회, 작물 수확 80회, 낚시 10회, 상점 판매 1회.
- 주간 퀘스트: 랭크 진행, 던전 클리어, 경매 참여, 마을 활동.
- 보상은 큰 돈보다 재료, 제작권, 던전 열쇠, 코스메틱 토큰 중심으로 둔다.

### 2. 컬렉션 / 도감

우선순위: 매우 높음.

Hypixel SkyBlock의 컬렉션은 단순 채집을 장기 성장으로 바꾼다. 아이템을
직접 모은 양을 추적하고, 일정 수량마다 레시피/보상/칭호를 연다.

우리 서버 적용:

- `LeeSeolCollection` 신규 플러그인 후보.
- 광물, 농작물, 낚시, 몹 드랍, 던전 보상, 보스 처치 컬렉션.
- 경매/상점 구매분은 컬렉션 진행에 넣지 않는다.
- 보상은 제작 레시피 해금, 소액 능력치, 칭호, 코스메틱, 던전 입장권.

### 3. 아이템 가치표와 희귀도 체계

우선순위: 매우 높음.

아이템을 더 넣기 전에 등급과 가치 기준이 필요하다.

우리 서버 적용:

- `COMMON`, `UNCOMMON`, `RARE`, `EPIC`, `LEGENDARY`, `MYTHIC` 같은 등급표를
  먼저 만든다.
- 각 등급마다 권장 획득처, 판매 가능 여부, 경매 가능 여부, 제작 재료 여부,
  던전 드랍 여부를 정한다.
- 던전 보상과 제작 재료를 이 표에 맞춰 추가한다.

### 4. 던전 반복 플레이 구조

우선순위: 높음.

Wynncraft와 Hypixel SkyBlock 모두 던전/레이드가 장기 목표 역할을 한다.

우리 서버 적용:

- `LeeSeolDungeon`에 던전 티어를 만든다: 초급, 중급, 상급.
- 입장 비용 또는 열쇠를 둔다.
- 클리어 상자는 고정 보상 + 확률 보상 + 컬렉션 진행을 함께 준다.
- 던전 보상은 바로 돈으로 팔리는 구조보다 제작/강화 재료 중심이 안전하다.

### 5. 경매 고도화

우선순위: 중간.

Hypixel식 경매/시장 구조는 강력하지만, 인원이 적은 서버에서는 너무 빨리
복잡해질 수 있다.

우리 서버 적용:

- 지금 당장 Bazaar식 실시간 시장은 보류한다.
- 먼저 `LeeSeolAuction`에 즉시구매, 판매 통계, 수수료, 경매 검색, 판매
  제한 아이템 목록을 추가하는 편이 현실적이다.
- 수수료는 돈 소모처가 되므로 밸런스에 도움이 된다.

### 6. 스킬 / 전문화

우선순위: 중간.

Hypixel의 스킬, Wynncraft의 클래스처럼 플레이어 성장 방향이 분명해야 한다.

우리 서버 적용:

- 지금 바로 복잡한 전투 클래스를 만들 필요는 없다.
- `LeeSeolJobs`를 확장해 광부, 농부, 낚시꾼 숙련도와 작은 패시브를 준다.
- 패시브는 돈 증가보다 편의성/확률/레시피 해금 중심이 안전하다.

### 7. 코스메틱 중심 장기 보상

우선순위: 중간.

MCC Island는 코스메틱, 배지, 이벤트 패스로 반복 플레이 이유를 만든다.

우리 서버 적용:

- 밸런스에 영향을 주지 않는 칭호, TAB 테마, 이펙트, 파티클, 로비 장식,
  프로필 배지를 장기 보상으로 둔다.
- 돈/장비 보상만으로 장기 플레이를 설계하면 인플레이션이 빨라진다.

## 지금 하지 않는 편이 좋은 것

- 대형 실시간 시장: 플레이어 수가 적으면 가격 조작과 빈 시장 문제가 크다.
- 자동 자원 생성기: 경제가 안정되기 전에는 Jobs 수익과 충돌한다.
- 복잡한 클래스 전투 시스템: 현재 Combat, Ranks, AdvancedEnchantments와
  충돌할 가능성이 높다.
- 대량 커스텀 아이템 추가: 가치표 없이 넣으면 밸런스 조정이 어려워진다.
- 리소스팩 대규모 재작업: ItemsAdder/pack 안정성이 이미 민감하므로 기능
  루프가 먼저다.

## 추천 개발 순서

1. `LeeSeolQuest`를 일일/주간 운영 패스로 확장.
2. 아이템 가치표 작성.
3. `LeeSeolCollection` 설계 또는 `LeeSeolQuest` 안의 컬렉션형 목표로 시작.
4. `LeeSeolDungeon` 보상/열쇠/상자 티어 설계.
5. `LeeSeolAuction` 즉시구매, 수수료, 검색, 통계 추가.
6. Jobs 숙련도와 소형 패시브 추가.
7. 코스메틱/배지/칭호 보상 연결.
